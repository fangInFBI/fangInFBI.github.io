char字符

解释几个char字符知识点：

码点：不同国家有自己语言，汉语，英语，俄语等，在计算机早期时代，这些语言在计算机中的编码方式是没有统一的，比如，美国的ASCII编码，俄罗斯的KOI，中国的GB等等。这些编码方式的不统一就会造成一些存储方面的问题。于是Unicode1.0就出现了，这是一种专门为了统一不同字符编码的所编制一种编码方式。但是在最初的时候，所有人都认为用8位数字就可以表示这个世界上所有的字符，字符的码点就是该字符的编码。

单元代码：不过后来由添加了大量汉字导致8位数字不够用了，随后出现了UTF-16编码，即使用16位数组表示字符，这16位的数字组合叫做单元代码。一些常用的字符使用一个单元代码就可以表示，但是一些辅助字符就需要一对单元代码进行表示。不过可以通过单元代码的范围分辨不同的字符。也就是说常用的字符的码点使用一个单元代码就可以表示，辅助字符的码点需要一对单元代码进行表示。char类型就是描述了UTF-16中一个单元代码。

String字符串

String数据类型并不是JAVA基本数据类型，而是在char数据类型的基础上实现的。在String类的源码实现中，通过char[]数组的形式定义数据对象，并且每次针对String字符串的操作（比如拼接，替代等）都是新建一个char[]数组对象，对该对象进行操作返回最后的结果。

字符串常量池

由于字符串的使用是非常频繁的，所以jvm为了减少内存的开销，提高代码的性能，使用了常量池的概念。也就是String对象在进行创建时，首先检查常量池里面是否存在这个字符串实例，如果存在，则直接返回这个字符串对象的地址。如果不存在，则首先在字符串常量池中实例化该字符串实例，接着将该实例的引用返回给创建的对象。字符串在创建之后就不会再改变（这里的改变是指的上面我们说的字符串的各种操作都是创建了一个新对象进行操作，常量池中原有字符串对象并没有改动），所以常量池中并不会出现两个相通的字符串对象。
